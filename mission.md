# WooTeGo

## 학습 목표
- 관련 함수를 묶어 클래스를 만들고, 객체들이 협력하여 하나의 큰 기능을 수행하도록 한다.
- 클래스와 함수에 대한 단위 테스트를 통해 의도한 대로 정확하게 작동하는 영역을 확보한다.
- 바둑 게임의 핵심 규칙(착수, 따내기, 자충수 등)을 객체 지향적으로 구현한다.

## 기능 요구 사항

### 바둑 게임
간단한 바둑 게임을 구현한다.

#### 게임 규칙
- 바둑판은 19x19 크기이다.
- 두 명의 플레이어(흑, 백)가 번갈아가며 착수한다.
- 흑이 먼저 시작한다.
- 한 번 놓은 돌은 다시 들 수 없다.
- 이미 돌이 있는 위치에는 착수할 수 없다.

#### 따내기 규칙
- 상대방 돌의 활로가 0이 되면 해당 돌(또는 연결된 돌 그룹)을 따낸다.
- 따낸 돌의 개수를 집계한다.

#### 자충수 규칙
- 착수 후 자신의 돌 그룹의 활로가 0이 되는 수는 둘 수 없다.
- 단, 상대방 돌을 따내서 활로가 생기는 경우는 가능하다.

#### 패(Ko) 규칙
- 직전 국면을 즉시 재현하는 착수는 금지된다.
- 한 수를 다른 곳에 둔 후에는 패를 다시 따낼 수 있다.

#### 게임 종료
- 양쪽이 연속으로 패스하면 게임이 종료된다.
- 게임 종료 시 점수를 계산하여 승자를 결정한다.

#### 점수 계산
- 한국 룰로 적용 : 사석을 집에 추가 및 백 덤 6.5집
- 계가 기능은 KataGo를 사용하여 별도로 구현 예정

### 입출력 요구 사항

#### 입력
- HTML, CSS, JavaScript를 사용한 프론트엔드로 구현
- 바둑판 클릭으로 착수 위치 입력
- 패스 버튼 클릭으로 패스 입력

#### 출력
- 바둑판 상태를 화면에 시각적으로 표시
- 현재 차례(흑/백) 표시
- 따낸 돌 개수 표시
- 게임 종료 시 승자 및 점수 표시

#### 예외 상황 시 에러 문구를 출력해야 한다.
- 에러 문구는 "[ERROR]"로 시작해야 한다. 프론트엔드에서 알림(alert 또는 toast)으로 표시
```
  [ERROR] 유효하지 않은 좌표입니다.
  [ERROR] 이미 돌이 있는 위치입니다.
  [ERROR] 자충수는 둘 수 없습니다.
  [ERROR] 패 규칙 위반입니다.
```

### 실행 결과 예시
- HTML, CSS, JavaScript를 사용한 프론트엔드 UI로 구현 예정

## 프로그래밍 요구 사항 1
- JDK 21 버전에서 실행 가능해야 한다.
- 프로그램 실행의 시작점은 Application의 main()이다.
- build.gradle 파일은 변경할 수 없으며, 제공된 라이브러리 이외의 외부 라이브러리는 사용하지 않는다.
- 프로그램 종료 시 System.exit()를 호출하지 않는다.
- 프로그래밍 요구 사항에서 달리 명시하지 않는 한 파일, 패키지 등의 이름을 바꾸거나 이동하지 않는다.
- 자바 코드 컨벤션을 지키면서 프로그래밍한다.
    - 기본적으로 Java Style Guide를 원칙으로 한다.

## 프로그래밍 요구 사항 2
- indent(인덴트, 들여쓰기) depth를 3이 넘지 않도록 구현한다. 2까지만 허용한다.
    - 예를 들어 while문 안에 if문이 있으면 들여쓰기는 2이다.
    - 힌트: indent(인덴트, 들여쓰기) depth를 줄이는 좋은 방법은 함수(또는 메서드)를 분리하면 된다.
- 3항 연산자를 쓰지 않는다.
- 함수(또는 메서드)가 한 가지 일만 하도록 최대한 작게 만들어라.
- JUnit 5와 AssertJ를 이용하여 정리한 기능 목록이 정상적으로 작동하는지 테스트 코드로 확인한다.

## 프로그래밍 요구 사항 3
- 함수(또는 메서드)의 길이가 15라인을 넘어가지 않도록 구현한다.
    - 함수(또는 메서드)가 한 가지 일만 잘 하도록 구현한다.
- else 예약어를 쓰지 않는다.
    - else를 쓰지 말라고 하니 switch/case로 구현하는 경우가 있는데 switch/case도 허용하지 않는다.
    - 힌트: if 조건절에서 값을 return하는 방식으로 구현하면 else를 사용하지 않아도 된다.
- Java Enum을 적용하여 프로그램을 구현한다.
- 구현한 기능에 대한 단위 테스트를 작성한다. 단, UI(System.out, System.in, Scanner) 로직은 제외한다.
